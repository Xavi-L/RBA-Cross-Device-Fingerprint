# 国内 Android 真机云测试平台补充调研（排除 Testin）

调研日期：2026-07-04

目标：在 WeTest 月度配额可能不足的情况下，寻找国内是否还有适合“上传自有 APK，并在真实 Android 设备上自动运行以采集设备 feature”的平台。这里排除 Testin，并优先考虑能做标准兼容测试、脚本兼容测试、云真机或批量自动化测试的平台。

## 结论

目前公开资料里，最值得继续验证的国内补充平台是 **百度 MTC**。它和 WeTest/Testin 的业务形态最接近，公开页面明确包含“脚本兼容测试”“远程真机调试”“云真机租用”“自动化测试”“专有云测试平台”等能力，并写明 Android TOP600、近千部在线机型、真实设备、报告日志和性能数据采集等信息。

阿里云 EMAS 现在更像移动研发、监控、推送、热修复、DevOps 平台。公开产品页虽然提到覆盖应用研发全生命周期和“测试”，但未看到类似 WeTest/MTC 这种可直接用于 Android TOP 机型批量兼容测试的公开设备池、机型列表或真机批量套餐。因此不建议把 EMAS 当作本项目的主采集平台。

各手机厂商应用市场的上架检测、隐私合规检测、兼容性预检，以及通用“云手机/云手游/云应用”类产品，也不建议作为主方案。它们可能能补充某些机型或系统生态，但通常不是面向公开批量选择真实终端的测试设备池；对本项目所需的真实设备型号覆盖、系统定制差异、传感器/硬件 feature 采集说服力较弱。

## 百度 MTC 可行性细节

百度 MTC 首页公开写明其产品服务包括云真机租用、脚本兼容测试、专家探索测试、智能硬件连通性测试、同业对比测试、专有云测试平台。脚本兼容测试包含在选定机型上安装、启动、Monkey、卸载，记录日志、截图、录像，并捕获 CPU、内存、流量等性能数据。

对本项目最关键的是两类入口：

1. **脚本兼容测试 / APP 兼容测试**
   - 公开页称覆盖市场主流机型，支持 Android TOP600 和 iOS TOP70。
   - 如果支持上传本项目 APK 并让 APK 主动回传 feature，则一次 TOP600 比 WeTest 400 台/月更适合做大规模采集。
   - 需要核实 TOP600 是“600 台设备运行一次”，还是“套餐权益中的一次测试任务”，以及失败重跑如何扣配额。

2. **远程真机调试 / 云真机租用**
   - 公开页称近千部机型实时在线，覆盖主流品牌和操作系统。
   - 套餐页列出半年 100 次/1000 分钟、年套餐 200 次/2000 分钟，以及年内不限时不限量的包年畅享。
   - 如果按“每台设备一次会话”采集，430 台至少需要 430 次会话，因此 100 次或 200 次套餐仍不够；如果能走脚本兼容测试批量任务，则性价比会高很多。

## 推荐执行路线

1. 先把百度 MTC 作为第一补充平台验证。
2. 登录后优先进入“脚本兼容测试”或“兼容测试”创建任务页，查看是否有设备选择器或 TOP600 设备池。
3. 如果能看到设备选择器，先尝试从网页静态资源或接口抓取设备列表，方法与 WeTest/Testin 相同。
4. 做一个 10-20 台的小规模 APK 试跑，确认 APK 能正常安装、启动、联网回传 feature，报告不会拦截后台网络。
5. 如果小样本成功，再购买或使用 Android TOP600 相关套餐；不要先买远程真机分钟包，因为分钟包的“会话次数”可能比总分钟数更早成为瓶颈。
