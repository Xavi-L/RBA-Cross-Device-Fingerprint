## 任务书

### I、毕业设计（论文）题目

基于三端融合设备指纹的移动端无感风控系统设计与实现

### II、原始资料、技术要求、选题意义和目标

本课题面向移动端 Hybrid App 无感风控场景，使用 Android Native、WebView 宿主和 Web 前端三类设备指纹数据。要求完成三端特征采集、session_id 会话合并、规则知识库构建、风险标签生成、轻量模型训练和端侧评分闭环。课题目标是实现 HybridGuard 原型系统，并验证跨层一致性特征对模拟器、无头环境、接口重放等风险识别的有效性。



### III、毕业设计（论文）工作内容

1. 调研设备指纹、移动端风控、WebView 安全和端侧推理相关文献。
2. 设计 HybridGuard 总体架构和三端数据采集方案。
3. 实现 Android 端采集、WebView/Web 探针和 FastAPI 后端会话合并。
4. 构建跨层语义规则知识库，生成风险标签并训练轻量评分器。
5. 实现端侧评分 App，完成消融实验、论文撰写和答辩材料准备。

## 开题报告

### 课题背景说明

移动端业务广泛承载登录、支付和账号操作，业务系统需要在尽量不打扰用户的情况下识别异常设备和异常环境。传统验证码和短信验证会影响体验，因此设备指纹和无感风控具有实际意义。

Hybrid App 中一次访问通常同时涉及 Android Native、WebView 宿主和 Web 前端。单一来源的指纹容易被伪造，而三端之间的设备型号、屏幕、UA、WebView 内核、JSBridge、传感器等信息可以相互印证。因此，本课题研究三端融合设备指纹采集、会话合并、跨层一致性分析和端侧风险评分。

### 毕设内容及目标

本课题拟实现 HybridGuard 原型系统。系统采集 Native、WebView 和 Web 三端特征，由后端按 session_id 合并并保存数据；离线阶段构建规则知识库，生成风险标签并训练轻量评分器；端侧阶段在 App 内完成本地采集、特征编码和随机森林风险评分。最终通过消融实验和分组交叉验证分析三端融合与跨层一致性特征的有效性。

## 总结

本毕业设计完成了 HybridGuard 三端融合设备指纹无感风控原型系统。系统实现了 Android Native、WebView 宿主和 Web 前端特征采集，能够按 session_id 完成会话合并，并结合规则知识库生成风险标签、训练轻量评分器和实现端侧评分闭环。

通过本课题，我熟悉了移动端采集、WebView 交互、后端数据处理、规则建模、模型训练和论文实验分析流程。课题仍存在真实设备样本有限、标签需要进一步复核、端侧性能测试不足等问题。总体来看，本设计完成了预定目标，并为后续扩展真实设备覆盖和优化端侧风控能力提供了基础。
