## 可统一替换的轻量规则

这些词不一定每处都必须改，但如果导师对语体比较敏感，可以做一轮全局搜索后择优替换。

| 表达 | 建议方向 | 说明 |
|---|---|---|
| 可以看到 / 可以看出 | 由……可见 / 结果表明 / 表明 | 实验分析和流程说明中更正式 |
| 这里 / 这样 | 本节 / 该流程 / 该策略 / 由此 | 减少课堂讲解感 |
| 变成 | 转为 / 转化为 / 呈现为 | 技术论文中更稳 |
| 做了 / 做出 | 实现了 / 设计了 / 构建了 | 避免工程笔记感 |
| 只看 / 只依赖 / 只记住 | 仅考察 / 仅依赖 / 对……形成记忆 | 更书面 |
| 排名靠前 | 重要性排名较高 / 权重较高 | 更适合实验结果表述 |
| 大量 / 一些 | 丰富的 / 若干 / 以下 | 减少泛泛而谈 |

## 不建议机械替换的表达

- “会”：正文里有不少“会引入、会记录、会高估”等表达，其中多数并不算口语；只有在语气过绝对时再改为“可能、将、可”。
- “可以”：技术论文中“可以”本身可接受，不必全改。建议只处理“可以看到、可以看出、这样可以把”等明显说明口吻。
- “当前”：多数用于说明数据集或系统状态，是准确表达，不属于口语化问题。
- “便于”：多数用于工程目的说明，语体可接受。
