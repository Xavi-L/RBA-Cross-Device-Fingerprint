## 本科生毕业设计（论文）任务书

### I、毕业设计（论文）题目

基于三端融合设备指纹的移动端无感风控系统设计与实现

### II、毕业设计（论文）使用的原始资料（数据）及设计技术要求/选题的意义、价值和目标

本课题面向移动端 Hybrid App 中的账号安全和无感风控需求，使用 Android Native、WebView 宿主和 Web 前端运行环境三类设备指纹数据作为原始资料。系统数据包括真实设备采集样本、扩充样本、高危模拟样本、后端合并会话数据、规则知识库标注结果以及消融实验结果。技术上要求完成三端特征采集、统一 session_id 会话对齐、后端数据接收与持久化、跨层语义规则构建、风险标签生成、轻量评分器训练和端侧评分闭环。

选题意义在于，单一 Web 指纹或单一 Native 指纹难以覆盖 Hybrid App 中设备、宿主容器和前端运行环境之间的复杂关系。通过三端融合，可以利用设备型号、系统版本、屏幕参数、WebView 内核、JSBridge、传感器、WebGL 和 UA 等信息之间的相互印证关系，提高对模拟器、无头浏览器、接口重放、宿主剥离和云真机环境的识别能力。课题目标是实现一个可运行的 HybridGuard 原型系统，并通过实验验证跨层一致性特征对移动端风险判断的有效性。

### III、毕业设计（论文）工作内容

1. 调研移动端无感风控、风险自适应认证、浏览器指纹、WebView 安全和端侧推理等相关研究，明确课题研究范围和系统定位。
2. 设计 HybridGuard 总体架构，划分 Android Native、WebView 宿主、Web 前端、后端服务、规则训练和端侧评分等模块。
3. 实现三端设备指纹采集与后端会话合并流程，完成原始 JSON 和训练 JSONL 数据持久化。
4. 构建跨层语义规则知识库，利用规则知识库辅助本地大模型完成离线风险标签生成。
5. 训练随机森林和 MLP 等轻量评分器，并将随机森林导出为 Android 可运行的 Java 推理代码。
6. 实现端侧评分 App，完成本地三端采集、65 维特征编码、风险分输出和评分摘要上报。
7. 设计并完成三端特征消融、跨层一致性消融、分组交叉验证和特征重要性分析，撰写毕业论文和答辩材料。

### 课题背景说明

随着移动互联网业务逐渐承载登录、支付、社交、内容发布和业务办理等高价值操作，业务系统需要在不明显打扰用户的前提下识别异常设备和异常访问环境。传统口令、短信验证码和图形验证码虽然能够提高安全性，但在低风险场景中会影响用户体验。因此，基于设备指纹和上下文信息的无感风控具有较高研究价值。

在 Hybrid App 场景中，一个业务会话通常同时涉及 Android Native 原生层、WebView 宿主容器和 Web 前端运行环境。Native 层可以反映设备型号、系统版本、物理屏幕、电池、传感器和调试状态；WebView 层可以反映宿主容器、JSBridge、WebView provider、安装来源和安全配置；Web 层可以反映 Navigator、逻辑屏幕、DPR、WebGL、Canvas、时区和算力挑战等信息。三类信息之间存在可相互印证的关系，局部字段可以被伪造，但多层特征之间的一致性更难长期保持。

现有研究多关注单端浏览器指纹、移动端认证或 WebView 安全配置，较少将 Native、WebView 和 Web 前端作为同一移动端会话中的互证信号进行系统建模。本课题拟设计并实现 HybridGuard 原型系统，重点研究三端设备指纹采集、会话合并、跨层语义规则、离线风险标注和端侧轻量评分闭环。

### 毕设内容及目标

本课题的主要内容是设计并实现一个面向 Hybrid App 的三端融合设备指纹无感风控系统。系统首先在 Android App 中采集 Native 原生特征，并通过 WebView 加载前端探针，采集 WebView 宿主信息和 Web 前端运行环境信息；随后由 FastAPI 后端根据统一 session_id 合并三端异步上报数据，并保存原始嵌套 JSON 与训练用 JSONL 数据。

在离线阶段，系统构建跨层语义规则知识库，用于描述屏幕一致性、UA 一致性、WebView 内核一致性、JSBridge 宿主真实性、传感器物理可信性、WebGL 渲染环境和攻击场景聚合等规则。基于规则知识库，本课题使用本地大模型辅助生成 risk_score 和 risk_reason，并将带标签数据用于随机森林和 MLP 等轻量评分器训练。

在端侧阶段，系统实现独立 riskapp 模块，在 App 内完成本地三端采集、特征编码、随机森林推理和风险摘要上报。实验部分将通过三端粗粒度消融、跨层一致性消融、分组交叉验证和特征重要性分析验证系统有效性。预期成果包括可运行原型系统、规则知识库、训练与实验脚本、端侧评分模块、实验图表和毕业论文。

## 总结

本毕业设计围绕“基于三端融合设备指纹的移动端无感风控系统设计与实现”展开，完成了从文献调研、系统设计、原型实现、规则知识库构建、离线风险标注、轻量模型训练、端侧评分集成到论文撰写和答辩准备的完整工作。系统实现上，HybridGuard 打通了 Android Native、WebView 宿主和 Web 前端三类设备指纹来源，使用统一 session_id 完成异步会话合并，并通过 FastAPI 后端保存原始数据和训练数据。规则与实验上，课题构建了跨层语义规则知识库，利用规则知识库辅助生成风险标签，并通过三端消融、跨层一致性消融、分组交叉验证和特征重要性分析验证系统有效性。
